The Testing Pyramid

Tests are organized in layers. Understanding this mental model helps you identify which test layer should catch specific bugs.

Key Insights

  • Unit tests are the foundation: Write many unit tests. They run fast, are easy to debug, and catch most bugs early.
  • Integration tests verify connections: These ensure your components work together correctly, especially at API boundaries.
  • E2E tests validate workflows: Use sparingly for critical user journeys. They're slow and can be brittle.
Mental Model Check: When a bug appears, ask yourself: "Which test layer should have caught this?" The answer reveals where your test coverage has gaps.
